## Why

`cc-session-reader` (forked from Mapleeeeeeeeeee/cc-session-reader to peterramsw/cc-session-reader) already compresses noisy Claude Code sessions into a readable, evidence-preserving transcript, but it only understands Claude Code and only produces a flat filtered view — there is no cross-agent support, no structured "what happened / what's next" handoff, and no way for a *new* session (Claude Code or Codex) to query old session state programmatically instead of re-reading a full transcript. Long sessions still die on context limits or process restarts with no cheap way to resume, and any resurfaced "tests passed" / "deployed" claim has to be manually re-verified by re-reading raw logs.

This change turns the fork into a multi-agent session context broker: Claude Code, Codex, and Antigravity sessions are detected through provider adapters, normalized into one event schema, and made available through deterministic filter/search/expand/inspect flows with no LLM dependency. A Local LLM distillation step can optionally produce a structured, evidence-referenced handoff artifact, but it is not required for the core product — a user with only Codex and no Local LLM must still be able to list, inspect, filter, search, and resume from evidence.

The real usage pattern is three agent apps (Claude Code, Codex, and Antigravity 2.0 — Google's standalone conversational multi-agent desktop app, not the older Antigravity IDE/VS Code fork) open at once, with only one actively driving work at any moment — the other two exist so that if the active one gets stuck, another can pick up the handoff and rescue the session. Antigravity 2.0 has both MCP support (stdio/SSE/Streamable-HTTP, config at `~/.gemini/config/mcp_config.json`) and its own Skill convention (`~/.gemini/skills/<name>/SKILL.md`, global install, frontmatter includes a `tools` field for MCP-tool access — notably the same `SKILL.md` filename as Claude Code, but a different install directory and a different frontmatter schema), so it needs a Skill package alongside Claude Code and Codex, not just MCP access. And — since each app spawns its own `cc-session serve-mcp` process over stdio — the evidence store must tolerate up to three concurrent server processes reading/writing the same `~/.session-context/` files without deadlocking or corrupting state. No container/daemon orchestration (no Docker) is needed for this; it's local file-level concurrency safety within a single Go binary.

## What Changes

- Add a `SessionProvider` adapter interface; keep the existing Claude Code parsing path (`internal/claudecodec`) behind it unchanged in behavior, add a new Codex adapter (`internal/codexcodec`) discovered from real local Codex CLI session files, and reserve the same adapter path for Antigravity session discovery/parsing once its real local format is verified.
- Introduce a normalized `SessionEvent` schema shared by both providers, with stable/deterministic event IDs, source-location traceability (path/line/byte offsets + content hash), and graceful handling of malformed lines and unknown event types (kept as `unknown`, never dropped).
- Extend (not rewrite) the existing deterministic filter (`internal/summarizer`, `internal/analyzer`) so error/warning/rollback/blocker/test/exit-code/git-state signals are guaranteed retained, and every compressed tool call keeps a structured evidence summary (never a bare "tool succeeded"), including for unknown tools.
- Add a deterministic secret-redaction pass (API keys, bearer/access/refresh tokens, passwords, cookies, client secrets, private keys, `.env` values, GitHub tokens, Anthropic/OpenAI-style tokens, JWTs, PEM blocks) applied before anything reaches the local LLM and before any filtered/handoff/evidence output, on by default.
- Add a file-based evidence store (`~/.session-context/<provider>/<session-id>/{manifest,normalized.jsonl,filtered.jsonl,evidence-index.json,handoff.json,handoff.md}`) with ID-based expansion, size-limited/truncation-flagged reads, and path-traversal / symlink-escape protection.
- Add a handoff JSON schema + validator (evidence-ID existence checks, unevidenced-claim demotion to `claims_requiring_reverification`, deployment/rollback conflict detection, branch/commit mismatch warnings, one JSON-repair retry, no silent empty handoff) and a renderer to `handoff.md` that always states the handoff is a derived artifact, not source of truth.
- Add an optional Local LLM distiller: OpenAI-compatible HTTP client (never hardcodes model/endpoint), phase-based chunking for context budgeting when filtered transcript exceeds configured limits, and a merge step that never invents evidence IDs. GB10/Qwen is one local deployment target, not a required dependency or hardcoded provider.
- Add an MCP stdio server (using a maintained Go MCP SDK) exposing `list_sessions`, `inspect_session`, `filter_session`, `create_handoff`, `get_handoff`, `search_session`, `expand_evidence`, `compare_context_size`, `verify_workspace` (read-only git checks only, restricted to `allowed_workspace_roots`, never mutates anything). The same server binary must be connectable from Claude Code, Codex, **and Antigravity** as independent stdio-launched processes running concurrently, with the evidence store safe against concurrent multi-process access (atomic writes, no cross-process deadlocks, no data corruption).
- Add new CLI subcommands to the existing `cc-session` binary sharing the same core as the MCP server: `list --provider`, `inspect`, `filter`, `handoff`, `search`, `expand <evidence-id>`, `verify-workspace`, `serve-mcp` — without breaking existing `list`/`read`/`context`/`stats`/`audit`/`expand`/`inject`.
- Add installable Skills for Claude Code, Codex, and Antigravity 2.0 (thin per-platform wrappers over shared content in `skills/common/`) implementing Resume / Close / Review-History workflows, where Resume always calls `verify_workspace` and never trusts handoff claims about tests/deployment/rollback without evidence or re-verification.
- Config additions: `session_sources.{claude_code,codex,antigravity}.roots`, `storage_root`, `allowed_workspace_roots`, optional `local_llm.{enabled,base_url,api_key,model,max_context,max_output_tokens,timeout_seconds}`, with matching env var overrides; core parse/filter/search/expand/stats functionality must work with zero dependency on Local LLM, Anthropic API keys, or any cloud OpenAI API.
- Documentation: `docs/architecture.md`, `docs/session-provider.md`, `docs/normalized-event-schema.md`, `docs/handoff-schema.md`, `docs/local-llm-distillation.md`, `docs/mcp-tools.md`, `docs/skills.md`, `docs/security.md`, `docs/upstream-sync.md`, and a README update with fork attribution, Codex/local LLM/MCP/Skill setup, and Resume/Close workflow walkthroughs.

## Capabilities

### New Capabilities
- `session-provider-abstraction`: `SessionProvider` interface, normalized cross-provider `SessionEvent` schema, Claude Code adapter kept behavior-compatible behind the interface, new Codex adapter, session discovery (Windows/Linux roots, env var + config overrides).
- `deterministic-filtering`: extended retention guarantees for risk signals (errors/warnings/rollbacks/blockers/tests/exit codes/git state), structured evidence summaries for every compressed or unknown tool call, stable evidence IDs.
- `secret-redaction`: deterministic detection/masking of credentials and secrets before local-LLM input and before any filtered/handoff/evidence output, with an explicit opt-in to disable.
- `evidence-store`: file-based per-session storage layout, ID-based evidence expansion with size limits and truncation flags, path-traversal/symlink protection, and safe concurrent access from multiple reader/writer processes (atomic writes, advisory locking, no deadlocks).
- `local-llm-handoff-distillation`: handoff JSON schema, validator (evidence existence, conflict/warning rules, JSON-repair-once), phase-based context-budgeted chunking + merge, OpenAI-compatible local LLM client, Markdown renderer with the required derived-artifact disclosure.
- `session-context-mcp-server`: stdio MCP server exposing session/handoff/evidence/workspace-verification tools, backed by the same core logic as the CLI, with workspace-root and evidence-size restrictions, and support for Claude Code, Codex, and Antigravity connecting as concurrent independent client processes ("one agent rescues another when it gets stuck" workflow).
- `cli-handoff-commands`: new `cc-session` subcommands (`list --provider`, `inspect`, `filter`, `handoff`, `search`, `expand <evidence-id>`, `verify-workspace`, `serve-mcp`) alongside unmodified existing commands.
- `cross-agent-skills`: shared Resume/Close/Review-History workflow content packaged as installable Claude Code, Codex, and Antigravity 2.0 Skills.

### Modified Capabilities
(none — this is a from-empty `openspec/specs/` baseline; the existing upstream CLI behavior is preserved as-is and re-described as part of `session-provider-abstraction`/`cli-handoff-commands` rather than modified)

## Impact

- Affected code: `cmd/cc-session/*` (new subcommand files), all of `internal/*` (new packages: `session/provider.go`, `codexcodec/`, `filtering/` or extensions to `summarizer`/`analyzer`, `evidence/`, `redaction/`, `handoff/`, `distiller/`, `mcp/`, `workspace/`), new `skills/` tree, new `docs/*.md`, `README.md`, `go.mod` (new dependencies: Go MCP SDK, HTTP client for OpenAI-compatible API).
- New runtime dependency (opt-in only): a locally-hosted OpenAI-compatible Local LLM endpoint for handoff generation only; all other functionality has no external dependency.
- New on-disk state: `~/.session-context/` evidence store (separate from Claude Code's/Codex's own session storage, which is never modified).
- No breaking changes to existing CLI commands or Claude Code session behavior; existing 280 upstream tests must continue to pass unmodified.
